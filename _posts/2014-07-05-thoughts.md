今天晚上，很随缘地遇到了内部做站点索引状态的网站，发现我这个站点被 block 了。因为正好有个 mail thread 讨论类似的问题，于是联系到质量组的一个同事。从他那得知，我这个站点之所以被 bing 抛弃了，原因就是我这个站点外链比较多——我囧。

吐槽质量算法是我瞬间的想法，但默默看了一下我的站点，啊，真的很久不更新，而且也不是什么有质量的文章（其实，之前的也不见得有什么质量）。关于我这个站点外链多，也不是说不过去，因为我一个 webget 放了我一些读过并且想分享的 RSS。同事很 nice 地帮我加个 trust ， 不过我也该看看我的站点了，真心要提高质量。

记得自己刚刚维护一个个人博客的时候，曾经了解过一下 SEO 的东西，至于我被识别成为垃圾站，可谓我 SEO 两个方面都不满足了，一个是站点内容不受机器欢迎，一个是站点也没高质量内容。

七月，工作满一年。这一年，说不忙碌不是，说很忙碌不算。某种意义上我成长了，但是能力的进步我自己不觉得满意。前几天想过类似的问题，不过自己没找到什么所以然。话说今天闪过一个念头，我学习新的东西变少了。自己建个博客，开始为了折腾，后来它更多地成为一种笔记，一种积累。写作的好处是，可以静静地把所思所想记录下来，有顺序，有逻辑，可以巩固自己所学的东西。但这一年来，觉得自己这方面变弱了很多。前段时间还尝试去做做题，但是发现自己有点刷不动了。忙碌的确是一方面，但是平时的确很想去休息，懒，疏于维护。平日工作的内容，也很少值得总结的。

巴拉巴拉列举一下最近所获，或许就是前天发现代码里面的 bug，因为 C# 没有声明抛出，一段老 code 里面直接抛出 Exception 一直没有被 catch 住，导致系统的 fall back 机制出现了问题。好吧，吐槽一下 C# 就像是一个不告诉你什么脾气的女孩，十分难对付。或许今天发现的一个问题也算吧，Level DB查询的cache 会对性能产生影响。我们有三个 replicas，如果我把测试对着三个负载平衡地跑，性能会不如对着一个跑的好。刚刚开始觉得这个很诡异，后来分析发现，是因为查询的模式会让 level db 把一个 block 加载到缓存中，后续的查询性能会提升。当对着多个运行的时候，让缓存的命中率变低了。上述两个事情，真的是很小很小的问题，一两句话说得差不多。或许我可以写一下或者总结一下，不过真心觉得只是一些很琐碎的东西。

想起一句话，不要因为忙碌而忙碌着，应该停下来，想想自己在干什么，要怎样做。工作一年的确是个特殊的时间，应该好好反思一下。这一年来，很少有我折腾的东西。前段时间老罗发部了他的锤子手机，关于有些观点认为各种吹是营销手段，但是有句话震撼了我的心——“我就是认真”。没用过锤子手机，不知道做得怎样，但这句话说到我有点惭愧，或许我缺这样一种，认真、耐心。
